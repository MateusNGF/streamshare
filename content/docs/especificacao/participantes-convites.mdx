# Participantes e Convites

Os Participantes são os membros que integram os compartilhamentos de streaming gerenciados por um Organizador. A plataforma oferece múltiplos mecanismos para adicionar e gerenciar estes membros, desde a inserção manual até ao ingresso autônomo via link de convite ou pelo Explorer.

---

## Registro de Participantes

O Organizador pode adicionar participantes manualmente, fornecendo os dados de contato de cada pessoa. Os campos disponíveis são:

- **Nome** (obrigatório);
- **Número de WhatsApp** (opcional, necessário para notificações);
- **CPF** (opcional, para identificação adicional);
- **E-mail** (opcional, permite vinculação automática a uma conta StreamShare).

### Regras de Unicidade

Para garantir que cada participante é representado de forma única dentro de uma conta:

- **CPF único por conta:** Não podem existir dois participantes com o mesmo CPF dentro da mesma conta.
- **WhatsApp único por conta:** Não podem existir dois participantes com o mesmo número de WhatsApp dentro da mesma conta.

Estas restrições evitam duplicações que poderiam gerar cobranças duplicadas ou notificações redundantes.

### Vinculação Automática a uma Conta StreamShare

Se o e-mail de um participante corresponder a um usuário já registrado na plataforma, o sistema estabelece automaticamente o vínculo entre os dois. A partir desse momento, o participante passa a ter acesso ao seu painel pessoal de faturas sem necessitar de fazer qualquer ação adicional.

### Herança de Verificação de WhatsApp

Quando um Organizador adiciona um participante com um número de WhatsApp, a plataforma verifica se esse número já foi validado por outro usuário do StreamShare. Se sim, a verificação é herdada automaticamente — dispensando o participante de passar pelo processo de validação novamente.

---

## Verificação do Número de WhatsApp

Para que as notificações automáticas por WhatsApp cheguem ao participante correto, o número de telefone deve estar verificado. O processo de verificação é simples:

1. O Organizador inicia a verificação a partir do perfil do participante no painel.
2. O sistema envia automaticamente um código numérico de 6 dígitos para o número de WhatsApp do participante.
3. O Organizador — ou o próprio participante, se tiver acesso — introduz o código recebido para concluir a verificação.

Só após este processo é que as notificações automáticas por WhatsApp ficam ativas para aquele participante.

---

## Remoção de Participantes

Um participante só pode ser removido da conta se não tiver **nenhuma assinatura ativa ou suspensa**. Se houver vínculos ativos com serviços, o sistema bloqueia a remoção e informa o Organizador quantas assinaturas precisam de ser canceladas antes de prosseguir.

A remoção de um participante não elimina o seu histórico financeiro — todas as cobranças e pagamentos passados ficam preservados através de um mecanismo de **remoção lógica** (*soft delete*).

## Convites por E-mail

O mecanismo de convite por e-mail permite ao Organizador convidar um usuário externo para integrar a sua conta no StreamShare. O usuário convidado recebe um e-mail com um link exclusivo e com prazo de validade de **7 dias**.

### Regras do Fluxo de Convite

- **Auto-convite Interditado:** O Organizador não pode enviar convites para o seu próprio endereço de e-mail.
- **Participante Já Ativo:** Se o e-mail já pertencer a um membro da conta, o convite é bloqueado.
- **Prevenção de Duplicados:** O sistema não permite enviar um novo convite se já existir um pendente e dentro do prazo para o mesmo destinatário.
- **Prazo de Expiração:** Os convites têm uma validade estrita de **7 dias**.
- **Vínculo a Serviço:** O convite pode ser vinculado a um serviço específico — ao aceitar, o usuário convidado é automaticamente inscrito nesse serviço.

### Aceitação do Convite

Quando o usuário aceita o convite:
1. Deve estar autenticado na plataforma (ou criar uma conta caso ainda não tenha).
2. É automaticamente registrado como participante na conta do Organizador.
3. Se o convite estiver vinculado a um serviço e o usuário ainda não tiver uma assinatura ativa para ele, a assinatura e a primeira cobrança são criadas imediatamente.

## Ingresso pelo Explorer (Solicitações Públicas)

O Explorer é a área pública da plataforma onde usuários autenticados podem descobrir serviços de streaming com vagas disponíveis.

### Fluxo com Aprovação Automática

Se o Organizador do serviço ativou a aprovação automática:
1. O usuário clica em "Solicitar Entrada" no Explorer.
2. A plataforma verifica se ainda existem vagas disponíveis.
3. O usuário é imediatamente registrado como participante e a assinatura é criada, sem qualquer intervenção do Organizador.

### Fluxo com Aprovação Manual

Se o serviço requer aprovação manual:
1. O usuário submete a sua "Intenção de Ingresso" pelo Explorer.
2. A solicitação fica pendente e os Administradores da conta são notificados.
3. A solicitação expira automaticamente após **48 horas** caso não seja aprovada ou recusada.
4. Se o Organizador aprovar, o usuário é registrado como participante e a assinatura é criada.
5. Se o Organizador recusar, o usuário recebe uma notificação informando que o seu pedido não foi aceito.
