# Convites e Entrada em Streaming

Qualquer novo membro do StreamShare entra numa partilha de streaming através de um **convite**. Existem dois tipos principais de convite.

## Tipos de Convite

| Tipo | Quem envia | Como funciona |
|---|---|---|
| **Link Direto (Público)** | Organizador, a partir do painel | Um link que pode ser partilhado livremente. Pode ser configurado para **uso único** ou **múltiplas utilizações**. |
| **Convite Nominal** | Organizador, informando o e-mail | Vinculado a um endereço de e-mail específico. Apenas o dono desse e-mail pode aceitar este convite. |

---

## O Fluxo de Aceitação de um Convite

Quando um usuário clica no link de convite, o sistema executa um conjunto de verificações para garantir que o processo é seguro e justo:

<Steps>
1. **Validação do link**: O sistema verifica se o link é legítimo. Caso contrário, o usuário recebe um aviso informando que o convite é inválido.
2. **Verificação do estado**: O sistema verifica se o link é de **Uso Único (Single-use)**. Se sim, e já tiver sido utilizado, o acesso é negado. Links de **Múltiplas Utilizações** permanecem ativos enquanto houver vagas e validade.
3. **Prazo de validade**: Os convites têm um tempo limitado para serem aceites. Se o prazo terminar, o usuário será informado de que o convite expirou.
4. **Disponibilidade de vagas**: O sistema verifica em tempo real se ainda existem lugares disponíveis, considerando todos os membros já inscritos. Se o limite for atingido, o sistema avisa que as vagas foram preenchidas.
5. **Aceitação (autenticação)**: Ao aceitar o convite, o sistema cria automaticamente o vínculo entre o participante e o serviço, além de gerar a primeira cobrança de forma segura, garantindo que o limite de vagas nunca seja ultrapassado.
</Steps>

---

## O que acontece após aceitar?

Imediatamente após a aceitação:
- A nova assinatura é criada com o estado inicial pendente.
- A primeira cobrança é gerada com a data de vencimento calculada.
- O Organizador é notificado sobre o novo participante.
- O Participante é encaminhado para as instruções de pagamento.

### Cálculo Proporcional (Pro-Rata)

Se o participante entrar durante um ciclo de faturamento já em curso, o sistema calcula automaticamente o **valor proporcional** aos dias de acesso que restam até ao próximo vencimento.

> **Exemplo:** Num serviço com vencimento no dia 1, se um participante entrar no dia 15, o sistema cobrará apenas o valor correspondente aos dias restantes do mês.

<Callout tipo="info">
O valor exato do rateio é apresentado de forma clara na tela de confirmação antes de o participante aceitar formalmente o convite.
</Callout>

---

## Quando as vagas esgotam

No momento em que o último lugar disponível é preenchido, os pedidos que ainda se encontravam em fila de espera são automaticamente cancelados e os respectivos usuários são notificados. Isso garante que ninguém fique a aguardar por uma vaga que já não existe.
